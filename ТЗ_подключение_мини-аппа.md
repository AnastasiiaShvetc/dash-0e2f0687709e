# ТЗ: подключение Оми к Telegram Mini App (дашборд недельных срезов)

## Контекст (что уже есть)

**Оми** — персональный ИИ-коуч (Telegram-бот `@echo_psycho_coach_bot`). Она каждый вечер (22:00 МСК) проводит рефлексию дня и **сохраняет данные дня** в файлы:
- `Дневник/ГГГГ-ММ-ДД.md` — один файл на день, со структурированным слоем данных (настроение, энергия, фокус, эмоции, победы по сферам и т.д.). Это фундамент.
- `Отчёты/Неделя_ГГГГ-Wnn.md` — недельные отчёты.

**Мини-апп** уже задизайнен и лежит в этой же папке:
- `index.html` — рабочий прототип (готов). Один файл, без внешних библиотек, подключён Telegram WebApp SDK, тянет данные из `data.json` через `fetch('data.json?v='+Date.now())` (антикеш), при сбое показывает заглушку.
- `data.json` — **рабочий файл, сейчас пустой** (`{"weeks": []}`). Сюда должны попадать реальные недели.
- `data.example.json` — демо-данные (4 недели, каждая помечена `"example": true`). Только образец структуры, **в прод не идут**.

## Задача

Связать Оми с мини-аппом так, чтобы:
1. Мини-апп визуализировал реальные данные пользователя, а не демо.
2. **Каждый день в 23:59 МСК** Оми пересобирала данные текущей недели (с учётом сегодняшнего дня) и записывала их в `data.json` на хостинге мини-аппа.
3. Дашборд при открытии/обновлении отражал их корректно.

Пользователь один (владелица), режим личный, без логина.

---

## Как работает поток данных

```
22:00  Оми проводит рефлексию → сохраняет Дневник/ГГГГ-ММ-ДД.md
23:59  Джоб: собрать текущую неделю из дневных файлов → обновить data.json
Открытие мини-аппа → fetch data.json → отрисовка
```

Ключевой принцип: **дашборд ничего не вычисляет и не хранит.** Всю агрегацию делает Оми и кладёт готовый `data.json`. Дашборд только рисует то, что в файле.

---

## Что делает ежедневный джоб (23:59 МСК)

1. Определить **текущую ISO-неделю** (понедельник–воскресенье), её id `ГГГГ-Wnn` и русский диапазон дат (напр. `13–19 июля`).
2. Собрать дневные файлы `Дневник/` за дни этой недели, которые уже прошли.
3. Посчитать агрегаты недели (см. схему ниже).
4. Найти в `data.json` запись недели с этим `id`:
   - если есть — **заменить** её (не плодить дубли, идемпотентно);
   - если нет — **добавить в конец** массива `weeks` (порядок строго хронологический — дашборд открывается на последней неделе).
5. Пересчитать `*_prev`-поля (см. «дельты»).
6. Атомарно записать `data.json`, предварительно провалидировав JSON и схему.

Джоб должен запускаться **после** вечерней рефлексии (22:00). Если за день рефлексии не было — обновляем неделю по имеющимся дням, сегодняшний день просто не добавляем (не выдумывать данные).

## Схема `data.json` (строго соблюдать имена полей)

```
{
  "weeks": [ WEEK, WEEK, ... ]   // хронологически; дашборд показывает последнюю
}
```

Объект WEEK:
```
{
  "id": "2026-W29",              // ISO-неделя
  "range": "13–19 июля",         // рус. диапазон
  "pobedy": {
    "total": 41,                 // всего задач за неделю (int)
    "tempo": "ровный",           // короткая словесная оценка темпа
    "hall_of_fame": ["...", ...],// 2–4 самых значимых достижения
    "coach": "..."               // комментарий коуча про динамику
  },
  "wheel": {
    "spheres": [                 // РОВНО 8, в этом порядке:
      {"name":"Работа","emoji":"💼","count":6},
      {"name":"Своё дело","emoji":"🚀","count":2},
      {"name":"Авторство","emoji":"✍️","count":4},
      {"name":"Отдых/заземление","emoji":"🧘","count":2},
      {"name":"Отношения","emoji":"❤️","count":0},
      {"name":"Здоровье","emoji":"🥦","count":3},
      {"name":"Яркость","emoji":"✨","count":1},
      {"name":"Саморазвитие","emoji":"🌱","count":5}
    ],
    "other": [{"name":"Быт","count":3}],  // «Всё остальное», НЕ входит в 8 сфер
    "coach": "..."
  },
  "compass": {
    "focus": 38,                 // % дел, ведущих к 4 главным направлениям (int)
    "focus_prev": 32,            // фокус прошлой недели
    "directions": [              // РОВНО 4:
      {"name":"Авторство","emoji":"✍️","score":4},
      {"name":"Своё дело","emoji":"🚀","score":2},
      {"name":"Путешествия","emoji":"✈️","score":1},
      {"name":"Отношения","emoji":"❤️","score":0}
    ],
    "avoidance_days": 3,         // сколько дней была «ловушка занятости»
    "avoidance_where": "учёба",  // куда убегала
    "coach": "..."
  },
  "state": {
    "mood": 6.4,                 // среднее настроение, шкала 0–10 (5 = нейтрально), 1 знак
    "mood_prev": 5.6,
    "energy": 5.9,               // средняя энергия, шкала 0–10, 1 знак
    "energy_prev": 5.3,
    "recovery_days": 2,          // дней настоящего восстановления (0–7)
    "mood_curve": [7,7,4,7,8,6,6],   // по дням недели, шкала 0–10
    "energy_curve": [7,7,2,6,7,6,6], // по дням недели, шкала 0–10
    "coach": "..."
  },
  "emotions": {
    "top": [                     // 3 главные эмоции
      {"name":"Тревога","count":5,"intensity":4,"trigger":"...","body":"грудь","valence":"neg"}
    ],
    "all": [                     // все эмоции недели (для «карты эмоций»)
      {"name":"Тревога","count":5,"valence":"neg"}
    ],
    "coach": "..."
  },
  "patterns": {
    "links": [                   // причинно-следственные связи
      {"cause":"прогулки (4 дня)","effect":"настроение выше на 2","dir":"pos"}
    ],
    "charge": [{"name":"прогулки","count":4}],  // что заряжало
    "drain":  [{"name":"созвоны без цели","count":3}], // что разряжало
    "coach": "..."
  },
  "conclusion": {
    "text": "...",               // главный вывод недели
    "step": "..."                // один маленький шаг на следующую неделю
  }
}
```

### Фиксированные значения (не менять)
- `valence`: только `"pos"` (позитивная), `"neg"` (негативная), `"neu"` (нейтральная). От неё зависит цвет: зелёный / красный / серый.
- `dir` в patterns.links: только `"pos"` / `"neg"`.
- Названия и эмодзи 8 сфер и 4 направлений — ровно как выше.

## Неочевидные детали (не потерять!)

1. **Дельты (`*_prev`).** Стрелки ↑/↓ в дашборде считаются как `текущее − prev`. Поэтому `mood_prev`, `energy_prev`, `focus_prev` = **финальные значения предыдущей недели** из `data.json`. Первая неделя без предыдущей — `prev` равен текущему (тогда покажется «без изменений»).

2. **Шкала настроения 0–10, нейтраль 5.** Графики рисуются относительно линии 5: участок выше 5 — зелёный, ниже — красный. Значения в `mood_curve`/`energy_curve` должны быть в шкале 0–10. Не присылать старую шкалу −5…+5.

3. **Частичная текущая неделя.** В понедельник–субботу неделя ещё не закончилась. В `*_curve` класть **только прошедшие дни** (например, во вторник — 2 значения), а не 7 нулей. Ноль нарисует ложный «обвал» в красную зону. Средние (`mood`, `energy`) считать по прошедшим дням.

4. **Новая неделя — новая запись.** В понедельник джоб создаёт **новый** объект недели и добавляет в конец, а не перезаписывает прошлую. Внутри недели (пн–вс) — обновляет одну и ту же запись по `id`.

5. **Идемпотентность.** Повторный запуск джоба в тот же день должен **заменять** запись недели, а не добавлять вторую и не удваивать счётчики.

6. **Демо-данные не смешивать.** В `data.json` (прод) не должно быть недель с `"example": true`. Месячные/любые агрегаты игнорируют всё, что помечено `example:true`. `data.example.json` — только справочник структуры.

7. **Часовой пояс — МСК** и для джоба (23:59), и для определения границ недели (ISO, пн–вс).

8. **Атомарная запись + валидация.** Писать во временный файл → провалидировать (JSON парсится, все обязательные поля, 8 сфер, 4 направления, valence из допустимых) → заменить `data.json`. Иначе кривой файл сломает дашборд. Держать бэкап предыдущей версии.

9. **Округление.** `mood`/`energy` — 1 знак после запятой; проценты и счётчики — целые.

10. **Кеш.** Дашборд уже добавляет `?v=timestamp`, но хостинг не должен агрессивно кешировать `data.json` (лучше отдавать с `Cache-Control: no-cache`), чтобы свежие данные подхватывались сразу.

---

## Хостинг и подключение к боту

1. `index.html` и `data.json` должны отдаваться **по HTTPS** (Telegram Mini App работает только по https; и `fetch` не работает с `file://`). Подойдёт любой статический хостинг / репозиторий-страница.
2. Оми (джоб бота) записывает `data.json` в ту же папку/бакет, откуда его отдаёт хостинг.
3. URL мини-аппа привязать к боту через **BotFather** (Mini App / кнопка меню). Пользователь открывает дашборд из бота.
4. Данные личные — доступ к хостингу закрыть; по желанию проверять `Telegram.WebApp.initData` на подлинность.

---

## Критерии готовности

- Открытие мини-аппа из бота показывает **реальную текущую неделю**, а не демо.
- В 23:59 МСК `data.json` автоматически обновляется данными за день; при следующем открытии дашборд их отражает.
- Переключение недель листает реальные накопленные недели; дельты и цвета графиков считаются корректно.
- Пустой старт (`weeks: []`) показывает аккуратную заглушку, а не сломанный экран.
